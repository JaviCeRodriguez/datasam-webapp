# DataSam WebApp

Aplicación web moderna para la gestión de datos y formularios, construida con Next.js 15, React 19 y TypeScript.

## 🚀 Inicio Rápido

¿Quieres contribuir al proyecto? **[Lee nuestra Guía de Desarrollo Local](./wiki/Guia-de-Desarrollo-Local.md)** para comenzar.

Para flujo completo de colaboración (fork, Supabase, envs, migraciones y uso de agentes IA), revisa la **[Guía de Colaboración](./wiki/Guia-de-Colaboracion.md)**.

La guía incluye:
- ✅ Requisitos previos e instalación
- ✅ Cómo ejecutar el proyecto localmente
- ✅ Flujo de contribución
- ✅ Uso responsable de herramientas de IA (Copilot, Cursor)
- ✅ Solución de problemas comunes

## 📋 Tecnologías

- **Framework**: [Next.js 15.2.6](https://nextjs.org/)
- **UI Library**: [React 19](https://react.dev/)
- **Language**: [TypeScript](https://www.typescriptlang.org/)
- **Styling**: [Tailwind CSS 4.1.9](https://tailwindcss.com/)
- **Components**: [Radix UI](https://www.radix-ui.com/)
- **Forms**: [React Hook Form](https://react-hook-form.com/) + [Zod](https://zod.dev/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Package Manager**: [pnpm](https://pnpm.io/)

## ⚡ Comandos Principales

```bash
# Instalar dependencias
pnpm install

# Ejecutar en modo desarrollo
pnpm dev

# Construir para producción
pnpm build

# Ejecutar en producción
pnpm start

# Verificar código con linter
pnpm lint
```

## 🤝 Contribuir

¡Las contribuciones son bienvenidas! Para contribuir:

1. **Lee la [Guía de Desarrollo Local](./wiki/Guia-de-Desarrollo-Local.md)** - Aprende cómo configurar el proyecto
2. **Toma o propone un issue** - Revisa los [issues abiertos](https://github.com/JaviCeRodriguez/datasam-webapp/issues)
3. **Haz fork del repositorio** - Crea tu propia copia
4. **Crea una rama** - `git checkout -b mi-funcionalidad`
5. **Haz tus cambios** - Implementa tu mejora o corrección
6. **Ejecuta el linter** - `pnpm lint`
7. **Haz commit** - `git commit -m "Descripción clara"`
8. **Envía un PR** - Crea un Pull Request explicando tus cambios

> ⚠️ **Importante**: Coordina tu trabajo a través de issues antes de empezar a codificar.
